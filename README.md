# ramy-plugins

A small [Claude Code](https://code.claude.com) plugin marketplace.

## Plugins

### living-docs (v0.3.0)

Self-maintaining, **agent-agnostic** project memory. Writes plain markdown into
`.agent-memory/` so the same memory is readable by Claude Code, Cursor, Copilot,
Codex, Gemini CLI, and humans.

Three deterministic hooks (no LLM, no network) plus one opt-in LLM consolidator:

| When | Script | What it does |
| --- | --- | --- |
| **SessionStart** | `notes-inventory.sh` | Surfaces where you left off: today's `.agent-memory/sessions/YYYY-MM-DD.md`, else the most recent prior session, else `.agent-memory/index.md`. (Falls back to legacy project-root notes files if `.agent-memory/` doesn't exist.) |
| **Stop** | `notes-autolog.sh` | Regenerates `.agent-memory/inventory/AUTO-DRAFT.md` — an idempotent listing of uncommitted files in the working tree. |
| **Stop** | `session-narrative.sh` | Creates/refreshes `.agent-memory/sessions/YYYY-MM-DD.md` with a deterministic skeleton: branch, today's commits, today's files (committed + uncommitted). Auto-derived block updates on every Stop; human-written sections are preserved. |
| **On demand** | `consolidate.sh` | **Opt-in.** Fills the `LLM-NARRATIVE` block of today's session log from git context (commits + diffs). Uses your existing Claude Code CLI auth by default; falls back to `$ANTHROPIC_API_KEY` or `$OPENAI_API_KEY`. Never auto-invoked. |

## Why `.agent-memory/`?

Plain markdown in a known per-repo location. Every modern agent can read it,
none of them need a special integration to write to it, and `grep` + `git log`
work the same as they always have. If you ever want semantic search on top,
drop the directory into [memsearch](https://github.com/zilliztech/memsearch),
[agentmemory](https://github.com/rohitg00/agentmemory), or whatever wins next —
the format is the moat.

Recommended companion: an `AGENTS.md` at the repo root (the cross-tool
[Agentic AI Foundation](https://agents.md/) spec) with thin `CLAUDE.md` /
`GEMINI.md` / `.cursor/rules/main.mdc` / `.github/copilot-instructions.md`
shims pointing to it. The living-docs plugin handles the *memory* layer;
AGENTS.md handles the *instructions* layer.

## Install

```bash
# one-time: register this marketplace
claude plugin marketplace add ramyazzam/claude-plugins

# install + enable the plugin
claude plugin install living-docs@ramy-plugins
```

Or commit the equivalent to a repo's `.claude/settings.json` so it travels to teammates and cloud sessions:

```json
{
  "extraKnownMarketplaces": {
    "ramy-plugins": { "source": { "source": "github", "repo": "ramyazzam/claude-plugins" } }
  },
  "enabledPlugins": { "living-docs@ramy-plugins": true }
}
```

## Per-repo setup

**Recommended:** create `.agent-memory/` with subfolders (the modern path):

```
.agent-memory/
├── README.md
├── index.md             # human-curated TOC
├── decisions/           # ADR-style notes
├── sessions/            # YYYY-MM-DD.md per session (auto-scaffolded by Stop hook)
└── inventory/           # AUTO-DRAFT.md (auto-regenerated by Stop hook)
```

The plugin detects this directory and writes the modern layout. No
configuration file required.

### Legacy / v0.2.0 mode

If `.agent-memory/` doesn't exist, the plugin falls back to the older
behaviour: edit a project-root notes file (`CHANGELOG.md` /
`conversation_notes.md` / `NOTES.md`). Configure via `.claude/livingdocs.json`:

| Key | Meaning |
| --- | --- |
| `notesFile` | Relative path to the changelog file. Auto-detects if omitted. |
| `contentPattern` | `grep -E` regex matched against changed file paths. Required for the Stop auto-draft in legacy mode; in modern mode it just filters the inventory. |
| `imageIdPattern` | Optional `grep -E` regex; trailing digits extracted as IDs. |
| `entryHeading` | Top-level entry heading marker (default `"## "`). |

Example:

```json
{
  "notesFile": "CHANGELOG.md",
  "contentPattern": "(^|/)src/.*\\.(ts|tsx|js|jsx)$",
  "imageIdPattern": "ISSUE-[0-9]+",
  "entryHeading": "## "
}
```

## Using `consolidate.sh`

Opt-in. Reads today's session file, sends git context to an LLM, fills the
`LLM-NARRATIVE` block. Idempotent — re-run any time to rewrite that block.

```bash
# Today
bash "$CLAUDE_PLUGIN_ROOT/scripts/consolidate.sh"

# A specific date
bash "$CLAUDE_PLUGIN_ROOT/scripts/consolidate.sh 2026-05-28"
```

Auth resolution order:

1. `claude` CLI on `PATH` → uses your existing Claude Code auth (zero config)
2. `ANTHROPIC_API_KEY` → direct Anthropic API call (Haiku 4.5 by default)
3. `OPENAI_API_KEY` / `LIVING_DOCS_LLM_API_KEY` → OpenAI-compatible endpoint
   (override model with `LIVING_DOCS_LLM_MODEL`, endpoint with
   `LIVING_DOCS_LLM_BASE_URL`)

## Requirements

`bash`, `git`, and `jq` on `PATH`. `curl` is required only if you use the
direct-API paths in `consolidate.sh` (option 2 or 3); the default `claude -p`
path doesn't need `curl`.
